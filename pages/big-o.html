📈 Big-O Notation

Week 2: Algorithm efficiency and asymptotic analysis

What is Big-O?

Big-O notation describes the upper bound of an algorithm's growth rate as input size approaches infinity.

Key Idea: Focus on dominant terms, ignore constants and lower-order terms.

Example: 2n³ + 5n² + 0.15n + 7 ≈ O(n³)

Why Not Use Timers?

  • Time varies based on CPU, memory, OS, network
  • Different inputs give different times
  • Can't express relationship between input and time

Big-O gives us a machine-independent way to measure efficiency.

Big-O Rules

Law of Addition (Sequential Code)

When operations happen one after another: O(f) + O(g) = O(max(f, g))

for (int i = 0; i < n; i++) { ... }   // O(n)
for (int j = 0; j < n; j++) { ... }   // O(n)
// Total: O(n) + O(n) = O(n)

Law of Multiplication (Nested Code)

When operations are nested: O(f) × O(g) = O(f × g)

for (int i = 0; i < n; i++) {         // O(n)
    for (int j = 0; j < n; j++) {     // O(n)
        ...
    }
}
// Total: O(n) × O(n) = O(n²)

Common Complexity Classes

Big-O Name Example Speed
O(1) Constant Array access by index ⚡ Fastest
O(log n) Logarithmic Binary search ⚡ Very fast
O(n) Linear Linear search, single loop ✓ Good
O(n log n) Linearithmic Merge sort, Quick sort ✓ Good
O(n²) Quadratic Bubble sort, nested loops ⚠️ Slow
O(2ⁿ) Exponential Recursive Fibonacci ❌ Very slow

Code Examples

O(1) - Constant

int getFirst(int[] arr) {
    return arr[0];  // Always 1 operation
}

O(n) - Linear

int sum(int[] arr) {
    int total = 0;
    for (int i = 0; i < arr.length; i++) {
        total += arr[i];  // n operations
    }
    return total;
}

O(log n) - Logarithmic

// Each iteration cuts problem in half
while (n > 1) {
    n = n / 2;  // log₂(n) iterations
}

O(n²) - Quadratic

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        // n × n = n² operations
    }
}

O(2ⁿ) - Exponential

// Naive recursive Fibonacci
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);  // Two calls each time!
}

How to Identify Complexity

Quick Rules:
  • No loops → O(1)
  • Single loop (0 to n) → O(n)
  • Loop cuts input in half each time → O(log n)
  • Nested loops → O(n²)
  • Recursive with 2 calls → O(2ⁿ)
