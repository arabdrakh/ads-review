🔍 Searching Algorithms

Week 7: Linear Search, Binary Search, and Hashing

Complexity Summary

Algorithm Time Complexity Requires Sorted?
Linear Search O(n) No
Binary Search O(log n) Yes
Hash Table O(1) avg / O(n) worst No

Linear Search

Check each element one by one until found or end reached.

Simple but slow: Works on unsorted arrays, but O(n) time. 
int linearSearch(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;  // Found at index i
        }
    }
    return -1;  // Not found
}

Binary Search

Divide sorted array in half repeatedly, eliminating half each time.

⚠️ Requires sorted array! Will give wrong results otherwise. 
// Iterative Binary Search
int binarySearch(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target) {
            return mid;  // Found
        } else if (arr[mid] < target) {
            left = mid + 1;   // Search right half
        } else {
            right = mid - 1;  // Search left half
        }
    }
    return -1;  // Not found
}

// Recursive Binary Search
int binarySearchRecursive(int[] arr, int target, int left, int right) {
    if (left > right) {
        return -1;  // Not found
    }
    
    int mid = left + (right - left) / 2;
    
    if (arr[mid] == target) {
        return mid;
    } else if (arr[mid] < target) {
        return binarySearchRecursive(arr, target, mid + 1, right);
    } else {
        return binarySearchRecursive(arr, target, left, mid - 1);
    }
}

How It Works

Searching for 7 in [1, 3, 5, 7, 9, 11, 13]:

Step 1: left=0, right=6, mid=3 → arr[3]=7 → Found!

Searching for 9:
Step 1: left=0, right=6, mid=3 → arr[3]=7 < 9 → go right
Step 2: left=4, right=6, mid=5 → arr[5]=11 > 9 → go left
Step 3: left=4, right=4, mid=4 → arr[4]=9 → Found!

Hashing for Search

Hash tables provide O(1) average-case search by mapping keys to array indices.

// Using HashMap
HashMap<String, Integer> map = new HashMap<>();
map.put("Alice", 25);
map.put("Bob", 30);

int age = map.get("Alice");  // O(1) average
boolean exists = map.containsKey("Bob");  // O(1) average
💡 Hash tables are fastest for search when keys are known, but worst case is O(n) with many collisions.
