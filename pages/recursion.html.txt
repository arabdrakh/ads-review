        <h1>🔄 Recursion</h1>
        <h2>Week 1: Function calls, stack, and recursive thinking</h2>

        <div class="section">
            <h2>What is Recursion?</h2>
            <p>Recursion is a programming technique where a <strong>function calls itself</strong> to solve smaller instances of the same problem.</p>
            
            <div class="key-point">
                <strong>Key Idea:</strong> Break a complex problem into simpler sub-problems until you reach a trivial case (base case).
            </div>
        </div>

        <div class="section">
            <h2>Function Call Stack</h2>
            <p>When a function is called, it's added to the <strong>call stack</strong>. The stack follows <strong>LIFO</strong> (Last-In-First-Out) principle.</p>
            
            <ul>
                <li>Each function call creates a new <strong>stack frame</strong></li>
                <li>The frame stores local variables and return address</li>
                <li>When function returns, its frame is popped from stack</li>
            </ul>

            <div class="warning">
                ⚠️ <strong>Stack Overflow:</strong> Too many recursive calls without reaching base case will crash the program!
            </div>
        </div>

        <div class="section">
            <h2>Two Essential Parts</h2>
            
            <table>
                <tr>
                    <th>Component</th>
                    <th>Description</th>
                    <th>Example (Factorial)</th>
                </tr>
                <tr>
                    <td><strong>Base Case</strong></td>
                    <td>Simplest case that can be solved directly (stops recursion)</td>
                    <td><code>if (n <= 1) return 1;</code></td>
                </tr>
                <tr>
                    <td><strong>Recursive Case</strong></td>
                    <td>Breaks problem into smaller subproblems</td>
                    <td><code>return n * factorial(n-1);</code></td>
                </tr>
            </table>
        </div>

        <div class="section">
            <h2>Example 1: Print Numbers (n to 1)</h2>
<pre>
void printNumbers(int n) {
    // Base case
    if (n == 0) {
        return;
    }
    
    // Print current number
    System.out.println(n);
    
    // Recursive call with smaller problem
    printNumbers(n - 1);
}

// printNumbers(5) outputs: 5 4 3 2 1
</pre>

            <div class="note">
                💡 The recursive call comes AFTER the print, so numbers print in descending order.
            </div>
        </div>

        <div class="section">
            <h2>Example 2: Factorial</h2>
            <p><strong>Mathematical definition:</strong> n! = n × (n-1) × (n-2) × ... × 2 × 1</p>
            <p><strong>Recursive definition:</strong> n! = n × (n-1)!</p>
            
<pre>
int factorial(int n) {
    // Base case
    if (n <= 1) {
        return 1;
    }
    
    // Recursive case
    return n * factorial(n - 1);
}

// factorial(5) = 5 * 4 * 3 * 2 * 1 = 120
</pre>

            <h3>Trace of factorial(4):</h3>
<pre>
factorial(4)
  → 4 * factorial(3)
      → 3 * factorial(2)
          → 2 * factorial(1)
              → return 1        // base case reached
          → return 2 * 1 = 2
      → return 3 * 2 = 6
  → return 4 * 6 = 24
</pre>
        </div>

        <div class="section">
            <h2>Common Pitfalls</h2>
            <table>
                <tr>
                    <th>Pitfall</th>
                    <th>Problem</th>
                    <th>Solution</th>
                </tr>
                <tr>
                    <td>Missing base case</td>
                    <td>Infinite recursion → Stack Overflow</td>
                    <td>Always define when to stop</td>
                </tr>
                <tr>
                    <td>No convergence</td>
                    <td>Problem doesn't get smaller</td>
                    <td>Ensure each call moves toward base case</td>
                </tr>
                <tr>
                    <td>Incorrect base case</td>
                    <td>Wrong answer or infinite loop</td>
                    <td>Test with smallest valid input</td>
                </tr>
            </table>
        </div>

        <div class="section">
            <h2>Recursion vs Iteration</h2>
            <table>
                <tr>
                    <th>Aspect</th>
                    <th>Recursion</th>
                    <th>Iteration</th>
                </tr>
                <tr>
                    <td>Termination</td>
                    <td>Base case</td>
                    <td>Loop condition fails</td>
                </tr>
                <tr>
                    <td>Memory</td>
                    <td>Uses stack (more memory)</td>
                    <td>Uses variables (less memory)</td>
                </tr>
                <tr>
                    <td>Readability</td>
                    <td>Often cleaner for complex problems</td>
                    <td>Usually simpler for basic loops</td>
                </tr>
                <tr>
                    <td>Performance</td>
                    <td>Function call overhead</td>
                    <td>Generally faster</td>
                </tr>
            </table>
        </div>
