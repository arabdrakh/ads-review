            <h3>🔢 Big-O Complexity (Fastest → Slowest)</h3>
            <p><code>O(1)</code> → <code>O(log n)</code> → <code>O(n)</code> → <code>O(n log n)</code> → <code>O(n²)</code> → <code>O(2ⁿ)</code> → <code>O(n!)</code></p>
            
            <table>
                <tr>
                    <th>Pattern</th>
                    <th>Complexity</th>
                </tr>
                <tr><td>No loops, array access</td><td>O(1)</td></tr>
                <tr><td>Loop halving each time</td><td>O(log n)</td></tr>
                <tr><td>Single loop 0 to n</td><td>O(n)</td></tr>
                <tr><td>Efficient sorts</td><td>O(n log n)</td></tr>
                <tr><td>Nested loops</td><td>O(n²)</td></tr>
                <tr><td>2 recursive calls (Fib)</td><td>O(2ⁿ)</td></tr>
            </table>
        </div>

                    <li>First In, First Out</li>
                    <li>Uses: BFS, scheduling</li>
                </ul>

        <div class="cheat-section">
            <h3>⚡ Key Formulas</h3>
            
            <h4>Heap (array, 1-indexed)</h4>
            <ul>
                <li>Left child: <code>2i</code></li>
                <li>Right child: <code>2i + 1</code></li>
                <li>Parent: <code>i / 2</code></li>
            </ul>

            <h4>Recursion</h4>
            <ul>
                <li>Must have: <strong>Base case</strong> + <strong>Recursive case</strong></li>
                <li>Missing base case → Stack Overflow</li>
            </ul>

            <h4>Dynamic Programming</h4>
            <ul>
                <li>DP = Recursion + Memoization</li>
                <li>Requires: Overlapping subproblems + Optimal substructure</li>
                <li>Naive Fibonacci: O(2ⁿ) → DP Fibonacci: O(n)</li>
            </ul>
        </div>

        <div class="cheat-section">
            <h3>⚠️ Common Exam Traps</h3>
            <ul>
                <li><strong>Binary Search</strong> → Array MUST be sorted!</li>
                <li><strong>Quick Sort worst case</strong> → O(n²) when pivot is always min/max</li>
                <li><strong>Dijkstra</strong> → Only works with non-negative weights</li>
                <li><strong>Hash Table worst case</strong> → O(n) with many collisions</li>
                <li><strong>DFS uses Stack</strong>, <strong>BFS uses Queue</strong></li>
                <li><strong>InOrder traversal of BST</strong> → gives sorted order</li>
                <li><strong>Merge Sort space</strong> → O(n), not O(1)!</li>
            </ul>
        </div>
