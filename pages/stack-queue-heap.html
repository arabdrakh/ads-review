📚 Logical Data Structures

Week 4: Stack, Queue, and Heap

Stack (LIFO)

Last-In-First-Out: The last element added is the first to be removed.

Think of: A stack of plates — you take from the top!

Operations (All O(1))

Method Description
push(item)Add item to top
pop()Remove and return top item
peek()Return top item without removing
isEmpty()Check if stack is empty
size()Return number of elements

Implementation (Array-based)

class MyStack<T> {
    private Object[] arr;
    private int top = -1;
    private int capacity;
    
    public MyStack(int cap) {
        capacity = cap;
        arr = new Object[capacity];
    }
    
    public void push(T item) {
        if (top == capacity - 1) {
            throw new RuntimeException("Stack Overflow");
        }
        arr[++top] = item;
    }
    
    public T pop() {
        if (top == -1) {
            throw new RuntimeException("Stack Underflow");
        }
        return (T) arr[top--];
    }
    
    public T peek() {
        if (top == -1) return null;
        return (T) arr[top];
    }
    
    public boolean isEmpty() {
        return top == -1;
    }
}
💡 Use cases: Function call stack, undo operations, expression evaluation, DFS

Queue (FIFO)

First-In-First-Out: The first element added is the first to be removed.

Think of: A line at a store — first come, first served!

Operations (All O(1))

Method Description
enqueue(item)Add item to back
dequeue()Remove and return front item
peek()Return front item without removing
isEmpty()Check if queue is empty

Implementation (LinkedList-based)

class MyQueue<T> {
    private Node<T> front;
    private Node<T> rear;
    
    public void enqueue(T data) {
        Node<T> newNode = new Node<>(data);
        if (isEmpty()) {
            front = rear = newNode;
        } else {
            rear.next = newNode;
            rear = newNode;
        }
    }
    
    public T dequeue() {
        if (isEmpty()) {
            throw new RuntimeException("Queue Underflow");
        }
        T data = front.data;
        front = front.next;
        if (front == null) rear = null;
        return data;
    }
    
    public boolean isEmpty() {
        return front == null;
    }
}
💡 Use cases: BFS, scheduling, buffers, print queue

Heap

A complete binary tree that satisfies the heap property.

Types

Type Property
Max Heap Parent ≥ Children (root is largest)
Min Heap Parent ≤ Children (root is smallest)

Array Representation

For node at index i:

  • Left child: 2i (or 2i + 1 if 0-indexed)
  • Right child: 2i + 1 (or 2i + 2 if 0-indexed)
  • Parent: i / 2 (or (i - 1) / 2 if 0-indexed)

Operations

Operation Complexity
getMax() / getMin() O(1)
insert() O(log n)
extractMax() / extractMin() O(log n)
heapify() O(log n)

Insert Algorithm

  1. Add new element at the end (last position)
  2. Compare with parent
  3. Swap if violates heap property
  4. Repeat until property satisfied or reached root

Extract Algorithm

  1. Remove root (return it)
  2. Move last element to root
  3. Heapify down (compare with children, swap with larger/smaller)
  4. Repeat until property satisfied or no children
// Heapify (Max Heap) - fixes violation at index i
void heapify(int[] arr, int n, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;
    
    if (left < n && arr[left] > arr[largest])
        largest = left;
    
    if (right < n && arr[right] > arr[largest])
        largest = right;
    
    if (largest != i) {
        swap(arr, i, largest);
        heapify(arr, n, largest);  // Recursive heapify
    }
}
💡 Use cases: Priority queues, heap sort, finding k largest/smallest elements
