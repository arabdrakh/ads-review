📦 Physical Data Structures

Week 3: Array, ArrayList, and LinkedList

Array

Arrays store elements of the same type in contiguous memory locations.

Key Properties:
  • Fixed size (static)
  • Fast access by index: O(1)
  • Elements stored sequentially in memory
// Declaration and initialization
int[] numbers = new int[5];        // Size 5, all zeros
String[] names = {"Alice", "Bob"}; // Size 2

// Access by index
int first = numbers[0];   // O(1)
numbers[2] = 10;          // O(1)

ArrayList

Dynamic array that can grow and shrink automatically.

ArrayList<Integer> list = new ArrayList<>();

list.add(10);       // Add to end: O(1) amortized
list.add(0, 5);     // Add at index 0: O(n) - must shift!
list.get(2);        // Access by index: O(1)
list.remove(0);     // Remove at index 0: O(n) - must shift!
⚠️ Disadvantage: Adding/removing at position 0 requires shifting all elements → O(n)

LinkedList

A series of nodes where each node contains data and a reference to the next node.

Node Structure

class Node<T> {
    T data;
    Node<T> next;
    
    Node(T data) {
        this.data = data;
        this.next = null;
    }
}

Types of LinkedList

Type Description
Singly-linked Each node points to next only
Doubly-linked Each node points to next AND previous
Circular Last node points back to first

LinkedList Operations

class LinkedList<T> {
    Node<T> head;
    Node<T> tail;  // Optional, but improves performance
    
    // Add to beginning: O(1)
    void addFirst(T data) {
        Node<T> newNode = new Node<>(data);
        newNode.next = head;
        head = newNode;
        if (tail == null) tail = newNode;
    }
    
    // Add to end (with tail): O(1)
    void addLast(T data) {
        Node<T> newNode = new Node<>(data);
        if (tail == null) {
            head = tail = newNode;
        } else {
            tail.next = newNode;
            tail = newNode;
        }
    }
    
    // Remove first: O(1)
    T removeFirst() {
        if (head == null) return null;
        T data = head.data;
        head = head.next;
        if (head == null) tail = null;
        return data;
    }
}

Complexity Comparison

Operation Array ArrayList LinkedList
Access by index O(1) O(1) O(n)
Add at end N/A (fixed) O(1)* O(1)†
Add at beginning N/A O(n) O(1)
Remove at beginning N/A O(n) O(1)
Search O(n) O(n) O(n)
Memory Less Less More (pointers)

* Amortized, † With tail pointer

When to Use Which?

Use ArrayList when:
  • You need fast random access by index
  • Most operations are at the end
  • You want less memory overhead
Use LinkedList when:
  • Frequent insertions/deletions at beginning or middle
  • You don't need random access
  • Size changes frequently
